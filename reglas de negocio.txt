Campos CONTACTOS
Telefono 
Nombre 
Apellidos 
Descripcion 
Correo
Favoritos
Bloqueados

TELEFONO:
-Tipo String
-Indice
-Obligatorio

NOMBRE:
-Tipo String
-Obligatorio
-Limite de 25 caracteres

APELLIDOS
-Tipo String
-Opcional
-Limite de 25 caracteres

Descripcion
-Tipo String
-Opcional
-Limite de 50 caracteres

CORREO
-tipo String
-Opcional
-Acepta varios(lista de correos)

FAVORITO
tipo booleano

BLOQUEADO
-Tipo booleano

Funciones
CREAR:
-no se puede crear un contacto con numTelef, ya existente para evitar duplicados
-no acepta un nombre vacio
-no acepta telefonos incompatibles con la extension indicada
-acepta varios correos o ninguno
-Nombre obligatorio y limitado por caracteres
-Descripcion opcional limitado por caracteres.
LEER:
-Devuelve una lista de todos los contactos guardados y su informacion.
-no se mostrarán los contactos bloqueados
-se indicará por cada registro si es favorito.
-devolver registros con Paginación indicada en la query
LEER Favoritos:
-retornará una lista de contactos favoritos
LEER Bloqueados:
-retornará una lista de contactos bloqueados
MODIFICAR:
-no se aceptará como Queryparametro un numTelef que no exista.
-los cambios se efectuarán sobre el numTelef indicado,
-los demas campos contendrán las mismas reglas.
ELIMINAR:
-no se aceptará como parametro un numTelef que no exista.
BUSQUEDA AVANZADA:
devolverá a partir de un texto(queryParameter indicado en la uri) num Telefono o nombre una lista de contactos.
-se deberá indicar que tipo de busqueda se quiere realizar por telefono o por nombre
-si el campo esta vacio obviamente se retornará una lista vacia.
